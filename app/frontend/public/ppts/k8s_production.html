Segment 1 — RBAC

Role-Based Access Control

🔑 Core Concepts

  • Role — permissions within a namespace
  • ClusterRole — cluster-wide permissions
  • RoleBinding — bind role to user/group/SA
  • ServiceAccount — identity for pods

📄 Example Role

kind: Role metadata: name: pod-reader namespace: prod rules: - apiGroups: [""] resources: ["pods", "logs"] verbs: ["get", "list", "watch"]
🏆 Least privilege: Give pods their own ServiceAccount with only the permissions they need. Never use the default ServiceAccount — it's too permissive.
Segment 2 — NetworkPolicy

NetworkPolicies — Zero Trust

🔒 Default Deny All

kind: NetworkPolicy metadata: name: default-deny-all spec: podSelector: {} # all pods policyTypes: - Ingress - Egress # no ingress/egress rules = deny all

✅ Allow Specific Traffic

kind: NetworkPolicy spec: podSelector: matchLabels: { app: db } ingress: - from: - podSelector: matchLabels: { app: api } ports: - port: 5432
⚠️ NetworkPolicies require a CNI that supports them (Calico, Cilium, Weave). Flannel does NOT enforce NetworkPolicies.
Segment 3 — Quotas

Resource Quotas & LimitRanges

📊 ResourceQuota (per namespace)

kind: ResourceQuota metadata: name: team-quota namespace: team-a spec: hard: requests.cpu: "4" requests.memory: 8Gi pods: "20"

📏 LimitRange (per pod/container)

kind: LimitRange spec: limits: - type: Container default: # applied if not set cpu: 500m memory: 256Mi max: cpu: "2" memory: 2Gi
Segment 4 — GitOps

GitOps & SRE Runbooks

🔄 GitOps with ArgoCD

  • Git = single source of truth
  • ArgoCD watches Git repo for changes
  • Automatically syncs cluster to desired state
  • Drift detection & alerting
  • Full audit trail via Git history

📋 SRE Runbook Essentials

  • Alert name & severity
  • Impact — what's broken for users
  • Symptoms — what you'll see in dashboards
  • Diagnosis steps — kubectl commands to run
  • Remediation — exact steps to fix
  • Escalation — who to page next
🏆 SRE Golden Signals: Latency · Traffic · Errors · Saturation (LETS). Define SLOs for each. Alert when SLO budget is at risk, not just when something breaks.
Knowledge Check

Final Quiz 🎯

1. What is the difference between a Role and a ClusterRole?
Role is namespace-scoped (only applies within one namespace). ClusterRole applies across the entire cluster — used for cluster-wide resources like nodes, PersistentVolumes.
2. You apply a "default deny all" NetworkPolicy. What happens to existing pod traffic?
All ingress and egress traffic is immediately blocked for matching pods. You must create explicit allow policies for any traffic you need (DNS on port 53, inter-service, etc.).
3. What does a LimitRange do that ResourceQuota doesn't?
LimitRange sets default and max limits per individual container/pod. ResourceQuota sets total resource budget for the entire namespace. LimitRange prevents any single pod from being a resource hog.
4. Name the 4 SRE Golden Signals.
Latency (how long requests take), Traffic (how many requests), Errors (rate of failed requests), Saturation (how full the system is — CPU, memory, queue depth).
