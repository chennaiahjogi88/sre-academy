Segment 1 — Why K8s?

What is Kubernetes?

K8s Solves

  • › Running containers at scale across many nodes
  • › Auto-healing: restart failed containers
  • › Self-scaling: add/remove pods based on load
  • › Rolling updates with zero downtime
  • › Service discovery and load balancing
  • › Secret and config management

Core K8s Objects

  • › Pod — one or more containers
  • › Service — stable network endpoint
  • › Deployment — manage Pod replicas
  • › Namespace — logical isolation
  • › ConfigMap / Secret — configuration
  • › PersistentVolume — storage
Segment 2 — Pods

Pods

What is a Pod?

  • › Smallest deployable unit in K8s
  • › Wraps one or more containers
  • › Containers share network (localhost) and storage
  • › Each pod gets a unique cluster IP
  • › Ephemeral — don't rely on pod IP persistence
  • › Sidecar pattern: main + logging/proxy container

Pod YAML

apiVersion: v1
kind: Pod
metadata:
  name: myapp
  labels:
    app: myapp
spec:
  containers:
  - name: app
    image: myapp:1.0
    ports:
    - containerPort: 3000
Segment 3 — Services

Services

ClusterIP
Default
Internal-only virtual IP. Used for pod-to-pod communication inside the cluster
NodePort
External Access
Exposes service on a static port (30000-32767) on every node. Good for dev/testing
LoadBalancer
Cloud LB
Provisions a cloud load balancer (ALB/NLB). Standard production internet-facing exposure
ExternalName
DNS Alias
Maps service to an external DNS name. Bridge to external services from within the cluster
Headless
clusterIP: None
No virtual IP — returns pod IPs directly. Used with StatefulSets for direct pod addressing
Ingress
HTTP Routing
Not a Service type, but routes HTTP/HTTPS traffic to services by host/path rules
Segment 4 — Deployments

Deployments

Deployment YAML

apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 3
  selector:
    matchLabels: {app: myapp}
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template: # Pod template here

Deployment Commands

# Apply manifest
kubectl apply -f deploy.yaml

# Check status
kubectl rollout status deploy/myapp

# Update image
kubectl set image deploy/myapp \
app=myapp:2.0

# Rollback
kubectl rollout undo deploy/myapp

# Scale
kubectl scale deploy/myapp --replicas=5
Quiz

K8s Basics Quiz ⚡

Q1. What is the smallest deployable unit in Kubernetes?
✅ A Pod — it wraps one or more containers that share a network and storage namespace.
Q2. Which Service type provisions a cloud load balancer?
✅ LoadBalancer — it calls the cloud provider API to create an external LB (like AWS NLB) and routes traffic to the pods.
Q3. What is a ReplicaSet?
✅ A controller that ensures a specified number of pod replicas are always running. Deployments manage ReplicaSets for you.
Q4. How does a Service find which pods to route traffic to?
✅ Via label selectors — the Service selects all pods matching its selector labels (e.g., app: myapp).
Q5. What does maxUnavailable: 0 in a RollingUpdate mean?
✅ No pods can be unavailable during the update — new pods must be ready before old ones are terminated. This achieves zero-downtime deployments.
