🔐 JWT Auth Flow (middleware/auth.js)
Extract Bearer token → jwt.verify(secret) → SELECT sessions WHERE jti + NOT revoked → Check is_active → req.user = {id, email, role}
Revocation: logout sets sessions.revoked=true using token jti · 24h expiry by default

Key Data Flows
🔑 Login Flow
1 POST /api/auth/login {email, pw}
2 bcrypt.compare(pw, hash)
3 jwt.sign(userId, email, role, jti)
4 INSERT sessions (jti, ip, ua, expires)
5 Return {token, user}
6 localStorage.setItem(token, user)
7 Redirect → portal.html
📊 Progress Tracking Flow
1 Student views ppts/class-X.html
2 ws.sendProgress(class_id, slide, total)
3 WS event: 'slide_progress'
4 UPSERT class_progress table
5 PUT /api/classes/:id/progress (REST backup)
6 portal.html fetches /progress/all on load
7 Progress bars rendered per class card
📢 Announcement Flow
1 Admin fills form in admin.html
2 POST /api/announcements
3 INSERT into announcements table
4 global.io.emit('announcement', data)
5 All WS clients receive push event
6 ws.js: showToast(title, msg, type)
7 Add to announcement list in portal
🎬 Video Upload & Stream
1 Admin: POST /api/recordings/upload
2 Multer saves → /app/uploads/
3 INSERT into recordings table
4 Students: GET /api/recordings
5 Click play → /stream/:filename
6 HTTP 206 range-request streaming
7 Browser plays video progressively
