PromQL Basics

Instant Vector Selector:

http_requests_total{job="api", method="GET"} # Matches all time series with these labels

Range Vector Selector:

http_requests_total[5m] # Last 5 minutes of data for matching series

Label Matchers: =, !=, =~ (regex), !~

Offset Modifier: Subtract time offset from range

metric offset 1h

Aggregation Operators

sum, avg, min, max, count:

sum(http_requests_total) # Total requests avg(temperature) # Average temperature

rate() & irate():

rate(requests_total[5m]) # Per-second rate irate(requests_total[5m]) # Instantaneous rate

by / without clause:

sum by(job) (http_requests_total) sum without(instance) (metric)

Key PromQL Functions

rate(counter[time]): Per-second increase

increase(counter[time]): Total increase

histogram_quantile(q, metric): Percentile

histogram_quantile(0.95, request_duration_seconds_bucket)

predict_linear(): Linear prediction

absent(): Alert if metric is missing

delta(), deriv(): Rate of change

Common Queries

CPU Usage %:

100 - (avg by(instance) rate(node_cpu_seconds_total{mode="idle"}[5m]) * 100)

HTTP Error Rate:

rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m])

Request Latency p99:

histogram_quantile(0.99, http_request_duration_seconds_bucket)

Recording Rules

Pre-compute expensive queries for reuse

groups: - name: 'recording_rules' interval: 30s rules: - record: job:http_requests:rate5m expr: sum by(job) rate(http_requests_total[5m])

Naming Convention: level:metric:operations

  • instance: per-instance metrics
  • job: aggregated by job

Alert Rules

Components: expr, for, labels, annotations

alert: HighMemoryUsage expr: memory_usage_percent > 90 for: 10m labels: severity: warning team: platform annotations: summary: "{{ $labels.instance }} high memory" description: "Memory at {{ $value }}%"

Routing to Alertmanager based on labels

Quiz: PromQL & Alerting

1. What does rate(requests_total[5m]) calculate?

The per-second rate of increase of the counter over the last 5 minutes. It accounts for resets and extrapolates to give you requests per second.

2. How would you write a query to get the 95th percentile of request latency?

histogram_quantile(0.95, http_request_duration_seconds_bucket)

3. What does sum without(instance) do versus sum by(job)?

sum without(instance) removes the instance label, summing across all instances. sum by(job) keeps only the job label, grouping by it. Result depends on your label structure.

4. Why use recording rules instead of writing queries in dashboards?

Recording rules pre-compute expensive queries at regular intervals. This reduces query load, speeds up dashboard rendering, and makes metrics available for alerting and other queries.

5. In an alert rule, what does the "for" parameter control?

The "for" parameter specifies the minimum duration the condition must be true before the alert fires. It prevents alerts from temporary spikes (e.g., "for: 5m" means alert after 5 minutes of the condition being true).
