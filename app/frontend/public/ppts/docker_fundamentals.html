Segment 2 — Architecture

Docker Architecture

Components

  • › Docker Daemon (dockerd) — background service managing containers
  • › Docker CLI — you type commands here
  • › containerd — OCI-compliant container runtime
  • › Docker Hub — default public registry
  • › ECR, GHCR — private registries

Key Objects

  • › Image — read-only template (blueprint)
  • › Container — running instance of an image
  • › Volume — persistent data storage
  • › Network — container communication
  • › Registry — image storage and distribution
Segment 3 — Images

Images & Layers

How Layers Work

Container Layer (R/W) — your changes
COPY . . → app code
RUN pip install -r requirements.txt
RUN apt-get install python3
FROM ubuntu:22.04 (base layer)

Layers are cached — only changed layers rebuild

Layer Caching Tips

  • › Put rarely changing layers first (FROM, RUN apt)
  • › COPY requirements.txt before COPY .
  • › One RUN per logical step = one cache layer
  • › Use .dockerignore to skip large files
  • › Multi-stage builds reduce final image size
Segment 4 — Dockerfile

Writing a Dockerfile

# Stage 1: Build
FROM node:18-alpine AS builder
WORKDIR /app
# Copy deps first for layer caching
COPY package*.json ./
RUN npm ci --only=production

# Stage 2: Runtime
FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/node_modules ./node_modules
COPY . .
EXPOSE 3000
USER node # don't run as root!
CMD ["node", "server.js"]
Segment 5 — Running Containers

Essential docker Commands

Build & Run

# Build image
docker build -t myapp:1.0 .

# Run container
docker run -d \
-p 8080:3000 \
-e NODE_ENV=prod \
--name myapp \
myapp:1.0

# Check logs
docker logs -f myapp

Inspect & Manage

# List running
docker ps

# Exec into container
docker exec -it myapp sh

# Inspect config
docker inspect myapp

# Stop & remove
docker stop myapp
docker rm myapp
Quiz

Docker Fundamentals Quiz ⚡

Q1. What is the difference between an Image and a Container?
✅ Image = read-only template (blueprint). Container = running instance of an image (adds a writable layer on top).
Q2. Why should you copy requirements.txt BEFORE copying your source code?
✅ Layer caching — if your code changes but requirements don't, Docker reuses the cached dependency layer, speeding up builds.
Q3. What flag runs a container in the background (detached)?
✅ The -d flag (detached mode). docker run -d myimage
Q4. Why should you avoid running containers as root?
✅ Security — if the container is compromised, the attacker would have root privileges on the host. Use USER in Dockerfile.
Q5. What does multi-stage build achieve?
✅ Smaller final images — build dependencies (compilers, test tools) stay in the build stage and are not copied to the runtime image.
