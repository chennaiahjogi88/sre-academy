Segment 1

VPC Peering

What is Peering?

  • › Private connection between two VPCs
  • › Traffic stays on AWS backbone (never public internet)
  • › Works cross-account and cross-region
  • › Not transitive — A-B, B-C ≠ A-C
  • › CIDR blocks must not overlap

Use Cases

  • › Share services across accounts (shared services VPC)
  • › Connect dev, staging, prod environments
  • › Cross-region data replication
  • › SaaS vendor connectivity
  • › Alternative: Transit Gateway for hub-and-spoke
⚠️ For many VPCs, use AWS Transit Gateway instead of peering — it provides transitive routing

VPC Endpoints

Gateway Endpoint
S3 & DynamoDB
Route table entry, free to use. Traffic to S3/DDB stays in AWS network
Interface Endpoint
Most AWS Services
ENI with private IP, powered by AWS PrivateLink. Hourly + per-GB cost
Use Case
No Internet Needed
Private subnet instances reach AWS APIs without NAT Gateway or IGW
Cost Saving
Avoid NAT Costs
Heavy S3 traffic via Gateway Endpoint = $0 vs expensive NAT data transfer fees
Security
Endpoint Policies
Attach IAM-style resource policies to control which resources can be accessed
Common Services
SSM, ECR, Secrets
EC2 System Manager, ECR pull, Secrets Manager — all via Interface Endpoints
Segment 3 — Flow Logs

VPC Flow Logs

What They Capture

  • › Accepted and rejected IP traffic
  • › Source/dest IP, port, protocol
  • › Bytes and packet count
  • › Can apply to VPC, subnet, or ENI level
  • › Sent to CloudWatch Logs or S3

SRE Use Cases

  • › Troubleshoot connectivity issues
  • › Detect port scanning / unauthorized access
  • › Audit security group rules
  • › Compliance and forensics
  • › Analyze traffic patterns

Segment 4 — Transit Gateway

AWS Transit Gateway

Why Transit Gateway?

  • › VPC Peering is non-transitive — mesh of N VPCs needs N(N-1)/2 peering connections
  • › TGW acts as a regional hub — each VPC attaches once
  • › Supports VPC, VPN, and Direct Connect attachments
  • › Cross-region peering via TGW Peering
  • › Supports up to 5,000 VPC attachments

Route Tables & Segmentation

  • › Each TGW has one or more route tables
  • › Attachments are associated to a route table
  • › Use separate route tables to isolate dev / prod
  • › Blackhole routes to block specific paths
  • › Shared services VPC → propagate to all route tables
10 VPCs with peering = 45 connections  |  10 VPCs with TGW = 10 attachments
Segment 5 — Security Groups

Security Groups — Advanced Patterns

SG Referencing (Chaining)

  • › Inbound rule can reference another SG as source
  • › Allows instances in the source SG to connect
  • › Works cross-account with the account ID prefix
  • › Example: ALB SG → App SG → DB SG (3-tier)
  • › Never need to update rules when IPs change

Best Practices

  • › Principle of least privilege — only allow needed ports
  • › Avoid 0.0.0.0/0 on port 22/3389 — use SSM Session Manager instead
  • › Use SG references for tier-to-tier rules, not CIDR blocks
  • › Tag SGs clearly — name, purpose, environment
  • › Max 60 inbound + 60 outbound rules per SG (adjustable)
  • › Max 5 SGs per ENI (network interface)
3-tier pattern: Internet → ALB-SG → ALB-SG → App-SG (port 8080) → App-SG → DB-SG (port 5432)
Segment 6 — Hybrid Connectivity

Direct Connect & Site-to-Site VPN

AWS Direct Connect

  • › Dedicated private network connection from on-prem to AWS
  • › Speeds: 1 Gbps, 10 Gbps, 100 Gbps
  • › Lower latency, consistent bandwidth vs internet
  • › Private VIF → access VPC resources
  • › Public VIF → access public AWS services (S3, etc.)
  • › Use Direct Connect Gateway for multi-VPC / multi-region

Site-to-Site VPN

  • › Encrypted IPSec tunnel over the public internet
  • › Two tunnels per connection (HA — different AZs)
  • › Connects Customer Gateway (on-prem) to Virtual Private Gateway (VPC)
  • › Or attach to Transit Gateway for multi-VPC access
  • › Setup in minutes — good backup for Direct Connect
  • › Throughput limited by internet quality
Production pattern: Direct Connect as primary + VPN as failover, both via Transit Gateway
Quiz

VPC Deep Dive Quiz ⚡

Q1. Is VPC Peering transitive?
✅ No. If VPC A peers with B, and B peers with C, A cannot talk to C through B. Use Transit Gateway for transitive routing.
Q2. Which endpoint type supports S3 with no cost?
✅ Gateway Endpoint — free to use, requires a route table entry. Ideal for S3 and DynamoDB.
Q3. Where can VPC Flow Logs be sent?
✅ CloudWatch Logs, Amazon S3, or Amazon Kinesis Data Firehose.
Q4. What must be true about CIDR blocks for VPC Peering?
✅ They must not overlap. Overlapping CIDRs prevent a peering connection from being established.
Q5. Which service replaces complex mesh peering for many VPCs?
✅ AWS Transit Gateway — acts as a hub, enabling transitive routing between many VPCs and on-premise networks.
