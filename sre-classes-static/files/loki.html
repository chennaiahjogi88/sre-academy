Segment 1 — Why Loki?

Why Loki Instead of ELK?

Loki Advantages

  • › Lightweight — indexes labels, not full text
  • › Low storage — uses gzip compression naturally
  • › Cost-effective — 10x cheaper than ELK
  • › Prometheus-native — same label model as Prometheus
  • › Grafana integration — native datasource support
  • › Horizontal scale — designed for microservices

When to Use Loki

  • › K8s clusters — pod logs at scale
  • › Label-based search — not full-text docs
  • › Metrics+Logs — unified Prometheus ecosystem
  • › Budget-conscious — want low ops cost
  • › Not ideal for: Complex text search (use ELK)
  • › Not ideal for: Ad-hoc log queries (structured only)
Segment 2 — Architecture

Loki Architecture

Data Flow

# Promtail (agent) → Loki (processing) → Storage (S3/GCS) → Grafana (query/viz)
Pod logs —[scrape]→ Promtail —[http]→ Distributor —[route]→ Ingester —[flush]→ S3
↓
Grafana —[LogQL]→ Querier —[read]→ S3 / In-memory cache

Distributor

  • • Entry point for log streams
  • • Validates format and labels
  • • Routes to ingesters
  • • Scales horizontally

Ingester

  • • Buffers logs in memory
  • • Builds index (labels only)
  • • Flushes to storage periodically
  • • Maintains recent-log cache

Querier

  • • Processes LogQL queries
  • • Reads from ingesters + storage
  • • Merges results
  • • Returns to Grafana

Segment 4 — LogQL Basics

LogQL — Query Language

Stream Selectors & Filters

# Stream selector (labels only)
{app="myapp", env="prod"}

# Filter expressions (pattern match)
{job="api"} |= "error" # contains "error"
{job="api"} != "debug" # excludes "debug"
{job="api"} |~ "timeout|refused" # regex match

# JSON parsing + filter
{pod="api-*"} | json | status="500"

# Pattern parsing
{job="nginx"} | pattern "<method> <path> <status>"
Segment 5 — LogQL Metrics

Convert Logs to Metrics with LogQL

Metric Aggregation

# Count errors per minute
rate({app="myapp"} |= "error" [5m])

# Sum bytes transferred by pod (requires |= parsing)
sum by (pod) ({job="api"} | json bytes_sent)

# Histogram of response latencies
histogram_quantile(0.95, rate({job="api"} | json latency_ms [1m]))

# Alert on high error rate
rate({env="prod"} |= "fatal" [1m]) > 10

Quiz

Loki & LogQL Quiz ⚡

Q1. What does Loki index by default?
✅ Labels only (like app, namespace, pod) — NOT full-text log content. This is what makes it lightweight and cheap.
Q2. What is Promtail?
✅ A log-scraping agent (DaemonSet in K8s) that reads pod logs from disk, parses them, adds labels, and ships them to Loki.
Q3. How do you find all ERROR logs in the prod environment?
✅ Use LogQL: {env="prod"} |= "ERROR" — the pipe filters log lines containing "ERROR".
Q4. Write a LogQL query to count errors per minute for the "api" job
✅ rate({job="api"} |= "error" [1m]) — this returns errors/second for the past 1 minute window.
Q5. Why is Loki better than ELK for Kubernetes?
✅ Loki is 10x cheaper (no full-text index), scales horizontally, integrates natively with Prometheus labels/Grafana, and matches K8s label patterns perfectly.
