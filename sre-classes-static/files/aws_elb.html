Segment 1 — Load Balancing

Why Load Balancing?

The Problem

  • › Single server = single point of failure
  • › Traffic spikes overwhelm one instance
  • › Deployments require downtime
  • › No way to scale horizontally

ELB Solution

  • › Distributes traffic across multiple targets
  • › Health checks — removes unhealthy targets
  • › Scales automatically with traffic
  • › Enables zero-downtime deployments
  • › Terminates SSL/TLS

Segment 2 — ALB

Application Load Balancer

ALB Features

  • › Operates at Layer 7 (HTTP/HTTPS/gRPC)
  • › Content-based routing (path, host, headers)
  • › /api/* → API target group
  • › /static/* → S3 or static servers
  • › WebSocket and HTTP/2 support
  • › Sticky sessions (cookies)

Target Groups

  • › EC2 Instances
  • › IP addresses (ECS tasks, Lambda)
  • › Lambda functions
  • › Health checks per target group
  • › Deregistration delay (draining)
🔀 Use ALB when: you need URL-based routing, WebSockets, or microservices architecture
Segment 3 — NLB

Network Load Balancer

NLB Features

  • › Layer 4 — TCP, UDP, TLS
  • › Ultra-low latency (~100ms vs ALB ~400ms)
  • › Millions of requests per second
  • › Static IP per AZ (Elastic IP support)
  • › Preserves source IP address

When to use NLB

  • › Gaming, IoT, real-time apps
  • › TCP/UDP protocols (not HTTP)
  • › Fixed IP addresses required by clients
  • › Extreme performance requirements
  • › TLS passthrough (end-to-end encryption)
ALB — Smart routing, Layer 7 | NLB — Raw speed, Layer 4 | GLB — 3rd party appliances
Segment 4 — Security Groups

Security Groups

Key Properties

  • › Virtual firewall at the instance level
  • › Stateful — return traffic auto-allowed
  • › ALLOW rules only — no DENY rules
  • › Applied to ENI (network interface)
  • › Multiple SGs per instance allowed
  • › Can reference other SGs as source

Segment 5 — NACLs

Network ACLs

NACL vs Security Group

  • › NACL = subnet level | SG = instance level
  • › NACL is stateless — must allow inbound AND outbound
  • › NACL supports both ALLOW and DENY rules
  • › Rules evaluated in order (lowest rule # first)
  • › Default NACL allows all traffic
  • › Good for blocking specific IPs (DDoS)

Quiz

ELB & Security Quiz ⚡

Q1. ALB operates at which OSI layer?
✅ Layer 7 — Application layer (HTTP/HTTPS). It can route based on URL paths and headers.
Q2. Are Security Groups stateful or stateless?
✅ Stateful — if you allow inbound traffic, the return outbound traffic is automatically permitted.
Q3. Can you add DENY rules to a Security Group?
✅ No. Security Groups only support ALLOW rules. Use NACLs if you need to explicitly deny traffic.
Q4. Which LB type gives you a static IP address per AZ?
✅ NLB — Network Load Balancer assigns a static IP (or Elastic IP) per Availability Zone.
Q5. NACLs are applied at the _____ level.
✅ Subnet level — NACLs control traffic entering and leaving the entire subnet.
