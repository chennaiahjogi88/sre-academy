Segment 1

VPC Peering

What is Peering?

  • › Private connection between two VPCs
  • › Traffic stays on AWS backbone (never public internet)
  • › Works cross-account and cross-region
  • › Not transitive — A-B, B-C ≠ A-C
  • › CIDR blocks must not overlap

Use Cases

  • › Share services across accounts (shared services VPC)
  • › Connect dev, staging, prod environments
  • › Cross-region data replication
  • › SaaS vendor connectivity
  • › Alternative: Transit Gateway for hub-and-spoke
⚠️ For many VPCs, use AWS Transit Gateway instead of peering — it provides transitive routing
Segment 2

VPC Endpoints

Gateway Endpoint
S3 & DynamoDB
Route table entry, free to use. Traffic to S3/DDB stays in AWS network
Interface Endpoint
Most AWS Services
ENI with private IP, powered by AWS PrivateLink. Hourly + per-GB cost
Use Case
No Internet Needed
Private subnet instances reach AWS APIs without NAT Gateway or IGW
Cost Saving
Avoid NAT Costs
Heavy S3 traffic via Gateway Endpoint = $0 vs expensive NAT data transfer fees
Security
Endpoint Policies
Attach IAM-style resource policies to control which resources can be accessed
Common Services
SSM, ECR, Secrets
EC2 System Manager, ECR pull, Secrets Manager — all via Interface Endpoints
Segment 3 — Flow Logs

VPC Flow Logs

What They Capture

  • › Accepted and rejected IP traffic
  • › Source/dest IP, port, protocol
  • › Bytes and packet count
  • › Can apply to VPC, subnet, or ENI level
  • › Sent to CloudWatch Logs or S3

SRE Use Cases

  • › Troubleshoot connectivity issues
  • › Detect port scanning / unauthorized access
  • › Audit security group rules
  • › Compliance and forensics
  • › Analyze traffic patterns

Quiz

VPC Deep Dive Quiz ⚡

Q1. Is VPC Peering transitive?
✅ No. If VPC A peers with B, and B peers with C, A cannot talk to C through B. Use Transit Gateway for transitive routing.
Q2. Which endpoint type supports S3 with no cost?
✅ Gateway Endpoint — free to use, requires a route table entry. Ideal for S3 and DynamoDB.
Q3. Where can VPC Flow Logs be sent?
✅ CloudWatch Logs, Amazon S3, or Amazon Kinesis Data Firehose.
Q4. What must be true about CIDR blocks for VPC Peering?
✅ They must not overlap. Overlapping CIDRs prevent a peering connection from being established.
Q5. Which service replaces complex mesh peering for many VPCs?
✅ AWS Transit Gateway — acts as a hub, enabling transitive routing between many VPCs and on-premise networks.
