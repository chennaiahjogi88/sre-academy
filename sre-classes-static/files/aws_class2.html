Segment 1 — AWS Compute

EC2 – Elastic Compute Cloud

🖥️ What is EC2?

  • Virtual server (VM) running in AWS cloud
  • You choose OS, CPU, RAM, storage
  • Full root/admin access to the instance
  • Pay only for what you use (per second)
  • Launch in minutes, scale instantly

⚙️ Key Components

  • Instance — The running VM
  • AMI — OS image used to launch it
  • EBS — Attached disk/storage
  • Security Group — Firewall rules
  • Key Pair — SSH access to instance
  • Elastic IP — Static public IP
💡 Analogy: EC2 is like renting a computer in Amazon's data center. You pick the specs, install what you want, and pay by the hour.

AMI – Amazon Machine Image

📸 What is an AMI?

  • A pre-built template (snapshot) for launching EC2
  • Contains: OS + pre-installed software + config
  • Stored in S3, referenced by AMI ID
  • Region-specific (can be copied cross-region)

🔁 AMI Types

  • AWS-provided — Amazon Linux, Ubuntu, Windows
  • AWS Marketplace — Vendor software (paid)
  • Community — Public, shared by others
  • Custom/Private — Built & owned by you

Segment 1 — EC2

EC2 Pricing Models

🕐 On-Demand

  • Pay per second, no commitment
  • Most flexible, highest cost
  • Good for: Dev, spiky workloads
Best for: Unpredictable traffic

📅 Reserved Instances

  • 1 or 3 year commitment
  • Up to 72% discount vs On-Demand
  • Convertible RI = flexible family
Best for: Steady, predictable workloads

⚡ Spot Instances

  • Bid on unused AWS capacity
  • Up to 90% cheaper than On-Demand
  • AWS can terminate with 2-min notice
Best for: Batch jobs, ML training, CI/CD
🏠 Savings Plans – Commit to $/hr spend for 1-3 yrs. Up to 66% off. More flexible than RIs.
🖥️ Dedicated Hosts – Physical server for compliance, licensing. Most expensive.

Auto Scaling Group (ASG)

📈 What is ASG?

  • Automatically adds/removes EC2 instances based on demand
  • Defines: Min / Desired / Max instance count
  • Uses Launch Template (or Launch Config) to spin up new instances
  • Works with Load Balancer to distribute traffic

📊 Scaling Policies

  • Target Tracking — e.g., keep CPU at 50%
  • Step Scaling — scale by steps when alarm fires
  • Scheduled — scale at specific times
  • Predictive — ML-based proactive scaling

Segment 2 — AWS VPC

VPC – Virtual Private Cloud

🌐 What is VPC?

  • Your own private network inside AWS
  • Logically isolated from other AWS accounts
  • You define IP range using CIDR notation
  • Spans all AZs within a Region
  • Default VPC exists per region (ready to use)

🧱 VPC Building Blocks

  • Subnets — Public & Private segments
  • Route Tables — Controls traffic routing
  • Internet Gateway (IGW) — Internet access
  • NAT Gateway — Outbound internet for private
  • Security Groups — Instance-level firewall
  • NACLs — Subnet-level firewall
💡 CIDR Example: VPC CIDR = 10.0.0.0/16 → gives you 65,536 IP addresses to divide into subnets

Segment 2 — AWS VPC

VPC Deep Dive – Key Concepts

🔗 VPC Peering

  • Connect 2 VPCs privately
  • Traffic stays on AWS backbone
  • Non-transitive (A↔B, B↔C ≠ A↔C)
  • Can be cross-account or cross-region

🔒 VPC Endpoints

  • Access AWS services without internet
  • Gateway endpoint: S3, DynamoDB (free)
  • Interface endpoint: Most services (PrivateLink)
  • Improves security & reduces cost

🌐 Transit Gateway

  • Hub to connect many VPCs & on-prem
  • Transitive routing (unlike peering)
  • Centralized routing management
  • Good for enterprise multi-VPC setups
VPC Flow Logs — Capture IP traffic metadata in your VPC. Stored in CloudWatch or S3. Essential for security audits & troubleshooting.
Bastion Host — A public EC2 used to SSH into private EC2 instances. Security best practice for admin access.
VPN & Direct Connect — VPN over internet to on-prem. Direct Connect = dedicated physical connection for high throughput & low latency.
Segment 3 — AWS ELB

Elastic Load Balancing (ELB)

⚖️ Why Load Balancing?

  • Distribute traffic across multiple EC2 instances
  • Prevent single point of failure
  • Health check → route only to healthy targets
  • Enable zero-downtime deployments
  • SSL/TLS termination at the load balancer

🔀 3 Types of AWS Load Balancers

  • ALB — Application LB (Layer 7 · HTTP/HTTPS)
  • NLB — Network LB (Layer 4 · TCP/UDP)
  • GWLB — Gateway LB (Layer 3 · Inline security)
(Classic LB is legacy — avoid for new projects)
💡 OSI Model Quick Ref: Layer 4 = Transport (TCP/UDP, IP+Port) · Layer 7 = Application (HTTP headers, paths, hostnames)
Segment 3 — AWS ELB

💡 Rule of Thumb: Default to ALB for web apps. Use NLB when you need extreme performance, static IPs, or non-HTTP protocols.
Segment 3 — AWS ELB

ALB – Advanced Features

🗺️ Content-Based Routing

  • Path-based: /api/* → API servers, /images/* → CDN
  • Host-based: api.site.com vs app.site.com
  • Header/Query: Route by User-Agent, query string
  • Weighted: 90% → v1, 10% → v2 (blue-green)

🎯 Target Groups

  • ALB routes to Target Groups (not instances directly)
  • Target group = set of EC2s / IPs / Lambdas
  • Health check per target group
  • Deregistration delay (draining): 300s default

Segment 4 — AWS Networking Security

Security Groups – Instance Firewall

🛡️ How Security Groups Work

  • Attached to EC2 instances (or ENI)
  • Stateful — return traffic allowed automatically
  • Only ALLOW rules (no deny rules possible)
  • Filter by: Protocol, Port range, Source IP/SG
  • Multiple SGs can be attached to one instance
  • Default: All inbound denied, all outbound allowed

NACLs vs Security Groups

FeatureSecurity Group (SG)Network ACL (NACL)
Scope Instance level (ENI) Subnet level
State Stateful — return traffic auto-allowed Stateless — must allow both directions
Rules Allow only Allow AND Deny
Rule Order All rules evaluated together Evaluated in numbered order (lowest first)
Default All inbound denied / all outbound allowed All inbound & outbound allowed (default NACL)
Use Case Primary layer of security, per-instance control Subnet-wide rules, extra deny layer
💡 Best Practice: Use Security Groups as your primary defense. Add NACLs for subnet-level blocking (e.g., block a malicious IP range). Think of SG = apartment lock, NACL = building security gate.

Segment 5 — Quiz ⚡

Rapid-Fire Quiz

Q1. What is the difference between a Public Subnet and a Private Subnet?
✅ Public subnet has a route to the Internet Gateway (IGW), allowing direct internet access. Private subnet has no IGW route — it uses a NAT Gateway for outbound-only internet access.
Q2. You need to route /api/* to API servers and /* to web servers using one load balancer. Which ELB type should you use?
✅ ALB (Application Load Balancer) — it supports path-based routing rules at Layer 7. NLB only works at Layer 4 and cannot inspect HTTP paths.
Q3. What is the key difference between Security Groups and NACLs?
✅ Security Groups are stateful (return traffic auto-allowed), operate at instance level, and only support Allow rules. NACLs are stateless, operate at subnet level, and support both Allow and Deny rules.
Q4. Your EC2 in a private subnet needs to download OS updates from the internet. What do you need?
✅ A NAT Gateway placed in a public subnet. Update the private subnet's Route Table to send 0.0.0.0/0 traffic to the NAT Gateway. The NAT GW handles the outbound internet call.
Q5. What are the 3 numbers defined in an ASG and what do they mean?
✅ Min — lowest number of instances (floor). Desired — target instance count at normal load. Max — highest number of instances ASG can scale to (ceiling).
