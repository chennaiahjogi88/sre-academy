Segment 1 — Full Stack Architecture

EKS Observability Design

Complete Data Flow

EKS Cluster (Prometheus scrapes metrics + Promtail scrapes logs)
↓
├─ Prometheus (metrics) → EBS PVC → TSDB
├─ Loki (logs) → S3 (backend storage)
├─ Alertmanager (rules) → Slack/PagerDuty
└─ Grafana (dashboard) ← reads from Prometheus + Loki

# All deployed via Helm charts in monitoring namespace

What Gets Monitored

  • › EKS cluster: API server, nodes, etcd
  • › Workloads: Pod CPU/memory, restart counts
  • › Network: Service latency, connection errors
  • › Storage: PVC usage, attach failures
  • › Logs: All pod logs indexed by labels

Components

  • › node-exporter - Node metrics (CPU, disk, network)
  • › kube-state-metrics - K8s object state
  • › prometheus-operator - Manage Prometheus via CRDs
  • › grafana - Dashboards + alerting UI
  • › alertmanager - Route + suppress alerts
Segment 2 — kube-prometheus-stack

Deploy Prometheus Stack via Helm

Installation & Customization

# 1. Add Prometheus community Helm repo
helm repo add prometheus-community \
  https://prometheus-community.github.io/helm-charts

# 2. Create custom values.yaml
prometheus:
  prometheusSpec:
    retention: 30d
    storageSpec:
      volumeClaimTemplate:
        spec:
          storageClassName: ebs-sc
          resources:
            requests:
              storage: 100Gi

# 3. Install
helm install kube-prom prometheus-community/kube-prometheus-stack \
  --namespace monitoring --create-namespace \
  -f values.yaml
Segment 3 — Persistent Storage

EBS Storage for Prometheus & Grafana

EBS CSI Driver Setup

  • › Prerequisite: EBS CSI add-on installed in EKS
  • › Provides ebs.csi.aws.com provisioner
  • › Create StorageClass for gp3 volumes
  • › Prometheus TSDB stores on EBS
  • › Data persists across pod restarts
  • › Use encrypted volumes in production

Storage Configuration

apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: ebs-sc
provisioner: ebs.csi.aws.com
parameters:
  type: gp3
  iops: 3000
  throughput: 125
  encrypted: "true"
Segment 4 — Loki on EKS

Deploy Loki with S3 Backend

Loki Stack Install

# 1. Add Grafana Helm repo
helm repo add grafana \
  https://grafana.github.io/helm-charts

# 2. Install loki-stack
helm install loki grafana/loki-stack \
  --namespace monitoring \
  --set loki.enabled=true \
  --set promtail.enabled=true \
  -f loki-values.yaml

S3 Configuration

loki:
  config:
    storage_config:
      aws:
        s3: s3://my-loki-bucket
        region: us-east-1
        sse_encryption: true
  schema_config:
    configs:
    - from: 2024-01-01
      store: boltdb-shipper
Segment 5 — IRSA & Production Readiness

IAM Roles for Service Accounts (IRSA)

IRSA Setup for Loki S3 Access

# 1. Create IAM role with trust policy (OIDC)
aws iam create-role --role-name loki-s3-access \
  --assume-role-policy-document '{...oidc...}'

# 2. Attach S3 policy to role
aws iam attach-role-policy --role-name loki-s3-access \
  --policy-arn arn:aws:iam::aws:policy/AmazonS3FullAccess

# 3. Annotate ServiceAccount in K8s
apiVersion: v1
kind: ServiceAccount
metadata:
  annotations:
    eks.amazonaws.com/role-arn: arn:aws:iam::123456:role/loki-s3-access

Scaling Prometheus

  • › Resource requests: 2 CPU, 8Gi memory typical
  • › Thanos: Multi-cluster metrics federation
  • › Cardinality limits: Prevent metric explosion
  • › Vertical Pod Autoscaler: Auto-adjust requests
  • › Sharding: Split scrape targets across Prometheus replicas

Production Checklist

  • › Enable pod disruption budgets (PDB)
  • › Set resource requests + limits
  • › Use IRSA for S3/CloudWatch access
  • › Enable encryption (EBS, S3)
  • › Configure retention policies
Quiz

EKS Observability Quiz ⚡

Q1. What does kube-prometheus-stack include?
✅ Prometheus (metrics), Grafana (dashboards), Alertmanager (alert routing), node-exporter, kube-state-metrics, and prometheus-operator for managing CRDs.
Q2. Why do you need EBS CSI driver for Prometheus?
✅ To provision persistent EBS volumes for Prometheus TSDB so metrics survive pod restarts. Without it, data is ephemeral and lost.
Q3. How does Loki store logs in S3?
✅ Promtail ships logs to Loki's distributor. Ingesters buffer and compress, then flush to S3 as chunks. Querier reads from S3 + cache via LogQL.
Q4. What is IRSA and why use it?
✅ IAM Roles for Service Accounts — lets Kubernetes pods assume AWS IAM roles without storing access keys. Loki pod gets S3 access via OIDC, not credentials.
Q5. How would you prevent Prometheus from consuming too much memory?
✅ Set cardinality limits (--metric-relabel-configs), enable scrape limits, use PVCs to offload to disk, or deploy Thanos for federation + downsampling.
