Segment 1 — VPC Basics

What is a VPC?

Virtual Private Cloud

  • Logically isolated network in AWS
  • You define the IP range (CIDR block)
  • Spans all AZs in a Region
  • Default VPC exists in every region
  • You can have up to 5 VPCs per region

Why VPC?

  • Complete control over network topology
  • Isolate workloads from the internet
  • Define inbound/outbound traffic rules
  • Connect to on-premise via VPN/DirectConnect
  • Foundation for all AWS networking
💡 CIDR example: 10.0.0.0/16 gives you 65,536 IP addresses to slice into subnets

Segment 3 — Routing

Route Tables & Internet Gateway

Route Table

  • Every subnet is associated with a route table
  • Rules decide where traffic goes
  • Local route (10.0.0.0/16) is always implicit
  • Public subnet: adds 0.0.0.0/0 → IGW
  • Private subnet: adds 0.0.0.0/0 → NAT GW

Internet Gateway (IGW)

  • Horizontally scaled, HA by default
  • Attach one IGW per VPC
  • Enables EC2 instances with public IPs to reach internet
  • Performs 1:1 NAT for Elastic IPs
  • Free to use — you pay for data transfer
📌 Key rule: Public subnet = subnet whose route table has 0.0.0.0/0 pointing to an IGW
Segment 4 — NAT

NAT Gateway

Why NAT?

  • Private subnet instances need outbound internet (updates, APIs)
  • But they must NOT be reachable from the internet
  • NAT Gateway sits in a public subnet
  • Translates private IPs to NAT GW's Elastic IP
  • Return traffic is allowed back automatically

NAT Gateway vs NAT Instance

  • NAT GW: managed by AWS, HA within AZ
  • NAT Instance: self-managed EC2 (old approach)
  • NAT GW: up to 100 Gbps bandwidth
  • Deploy 1 NAT GW per AZ for HA
  • Cost: ~$0.045/hr + data transfer fees
Segment 5 — Architecture

3-Tier VPC Architecture

PUBLIC SUBNET
10.0.1.0/24
Load Balancers (ALB/NLB), NAT Gateways, Bastion Host. Has route to IGW. Receives internet traffic.
PRIVATE SUBNET
10.0.2.0/24
App servers, EKS worker nodes, Lambda. No direct internet access. Outbound via NAT Gateway. Security Group allows inbound only from ALB SG.
DATA SUBNET
10.0.3.0/24
RDS, ElastiCache, Redshift. No outbound internet. Security Group allows inbound only from App SG (port 5432/6379). Most restricted tier.
Best practice: Replicate all 3 tiers across at least 2 AZs for high availability
Segment 6 — Security

Security Groups vs NACLs

Security Groups

  • › Applied at the instance/ENI level
  • › Stateful — return traffic auto-allowed
  • › ALLOW rules only — no DENY
  • › Can reference other SGs as sources
  • › Up to 5 SGs per ENI
  • › First line of defense for instances

Network ACLs (NACLs)

  • › Applied at the subnet level
  • › Stateless — must allow both inbound AND outbound
  • › ALLOW and DENY rules — numbered priority
  • › Rule 100 processed before rule 200
  • › Default NACL allows all traffic
  • › Use for: blocking specific IPs at subnet level
Segment 7 — Quiz

VPC Quick Quiz ⚡

Q1. What makes a subnet "public"?
✅ Its route table has a route to an Internet Gateway (0.0.0.0/0 → igw-xxxx).
Q2. How many IGWs can you attach to a VPC?
✅ One. A VPC can only have one Internet Gateway attached at a time.
Q3. Where does a NAT Gateway live?
✅ In a public subnet — it needs internet access to forward traffic outbound for private instances.
Q4. What is the default VPC CIDR block AWS assigns?
✅ 172.31.0.0/16 — AWS creates a default VPC with this CIDR in every region.
Q5. Can a subnet span multiple Availability Zones?
✅ No. A subnet exists in exactly one AZ. You create multiple subnets across AZs for HA.
