Grafana Overview

What is Grafana?

  • Open-source visualization & monitoring platform
  • Data source agnostic (supports 100+ integrations)
  • Create interactive, shareable dashboards
  • Dashboards stored as JSON models (version control friendly)

Supported Data Sources: Prometheus, Loki, Jaeger, Elasticsearch, CloudWatch, Datadog, InfluxDB, MySQL, PostgreSQL

Data Sources & Configuration

Adding Prometheus:

  • Administration > Data Sources > Add new
  • Type: Prometheus
  • URL: http://prometheus:9090
  • Auth: Basic auth, TLS, custom headers

Provisioning: Configure via YAML for IaC

apiVersion: 1 datasources: - name: Prometheus type: prometheus url: http://prometheus:9090 access: proxy

Panel Types

Time Series: Line/bar charts over time (most common)

Stat (Big Number): Single value with gauge/sparkline

Gauge: Radial gauge showing current value

Bar Chart: Categorical data comparison

Table: Tabular query results

Heatmap: 2D density visualization

Logs Panel: Display logs from Loki

Dashboard Variables

Template Variables: Reusable dynamic selectors

$namespace, $pod, $job # Used in queries: {namespace="$namespace"}

Query Variables: Fetch options from Prometheus

query: label_values(up, job) # Populate $job with all unique job values

Advanced: Chained variables, repeat panels per value

Repeat: Generate a panel for each $pod

Quiz: Grafana Dashboards

1. What is the primary advantage of using template variables in Grafana?

Template variables allow you to create reusable, parameterized dashboards. A single dashboard can serve multiple namespaces, pods, or environments by changing the variable value instead of creating separate dashboards.

2. Which panel type would you use to show current memory usage with a visual gauge?

Gauge panel - Shows a single value with a radial gauge. You can also use Stat panel with gauge mode for more customization.

3. How can you make a Grafana dashboard portable and version-controlled?

Export the dashboard as JSON and store it in git. Dashboards are stored as JSON models, making them perfect for version control, CI/CD pipelines, and infrastructure-as-code approaches.

4. What is the difference between a static data source URL and provisioning?

Static: Manually add data sources via UI. Provisioning: Define data sources in YAML files for automated, reproducible setup via GitOps. Provisioning is preferred for production.

5. How would you use "Repeat" panels in a Grafana dashboard?

Set a panel's repeat field to a variable (e.g., $pod). Grafana will automatically create a copy of that panel for each value in the variable. Useful for showing metrics for each pod in a list.
