Prometheus Architecture

Pull-based Model: Prometheus actively scrapes targets

# Prometheus Server pulls metrics Prometheus Server → scrapes → /metrics endpoints ↓ TSDB (Time Series Database) ↓ Rules Engine + Alertmanager

Components:

  • TSDB: Local storage for time series
  • Rules Engine: Recording & alerting rules
  • Alertmanager: Alert routing & grouping
  • Pushgateway: For short-lived jobs

Data Model

Format: metric_name{label="value"} timestamp value

Example:

http_requests_total{job="api", method="GET"} 1645678901000 1234

Vector Types:

  • Instant Vector: Single value per series at a point in time
  • Range Vector: Time range of values [5m], [1h]

Metric Types

Counter: Monotonically increasing

http_requests_total{path="/api"} 5234 # Only increases, useful for rate() calculations

Gauge: Can increase or decrease

memory_usage_bytes 2147483648 # Up and down, shows current state

Histogram & Summary: Distribution of values

  • Histogram: Buckets for percentiles
  • Summary: Pre-calculated quantiles

Prometheus Configuration

prometheus.yml:

global: scrape_interval: 15s evaluation_interval: 15s scrape_configs: - job_name: 'prometheus' static_configs: - targets: ['localhost:9090'] - job_name: 'kubernetes' kubernetes_sd_configs: - role: 'pod'

Exporters & Instrumentation

Node Exporter: Host metrics (CPU, disk, network)

kube-state-metrics: Kubernetes object metrics

Blackbox Exporter: Probe endpoints (HTTP, DNS, TCP)

Application /metrics: Built-in with client libraries

# Go example import "github.com/prometheus/client_golang/prometheus" counter := prometheus.NewCounter(prometheus.CounterOpts{ Name: "requests_total", })

Alerting Rules

Alert Rule YAML:

groups: - name: 'app_alerts' rules: - alert: HighErrorRate expr: rate(errors_total[5m]) > 0.1 for: 5m labels: severity: critical annotations: summary: "High error rate detected" description: "{{ $value | humanizePercentage }} errors"

Quiz: Prometheus

1. Why does Prometheus use a pull-based scraping model instead of push?

Pull-based allows Prometheus to control scrape timing, detect if a target is down (no response), and requires no agent configuration on targets. It's also simpler for service discovery.

2. What is the difference between a Counter and a Gauge metric?

Counter only increases (monotonic) and resets on restart - good for counts. Gauge can go up or down - good for current state like memory usage or temperature.

3. Which exporter would you use to monitor Kubernetes object states?

kube-state-metrics - Exports metrics about Kubernetes objects like Deployments, Pods, StatefulSets, etc.

4. In an alerting rule, what does the "for" field control?

The "for" field specifies how long a condition must be true before the alert fires. This prevents flaky alerts from temporary spikes (e.g., "for: 5m").

5. What is the purpose of the Pushgateway?

Pushgateway allows short-lived jobs to push their metrics before they exit, since Prometheus won't be able to scrape them. Essential for batch jobs, cron tasks, and one-off processes.
