Segment 3 — EC2 Optimization

EC2 Cost Optimization

Pricing Models

On-Demand: Pay per hour, no commitment. Use for unpredictable workloads.
Reserved Instances (RI): 1-3 year commitment. Up to 72% discount. Use for baseline load.
Savings Plans: Flexible across instance families. Up to 72% discount. Better than RI.
Spot Instances: Interruptible. Up to 90% discount. Use with fallback strategy.

Right-Sizing Strategy

  • › Use Compute Optimizer recommendations
  • › Start large, right-size downward
  • › Monitor actual CPU/memory over 2 weeks
  • › Graviton (ARM) 20-40% cheaper than x86

Instance Size Mix

  • › Baseline (RIs) — stable load
  • › Scaling (On-Demand) — predictable spikes
  • › Burst (Spot) — unpredictable peaks

Kubernetes Cost Optimization

  • › Correct requests = efficient scheduling

  • › Under-request → poor performance

Segment 5 — Spot Instances

Spot Instances in K8s

Spot Best Practices

Diversify instance types: Use 6-10 different families. Reduces correlation of interruptions.
Graceful termination: 2-min warning before eviction. Implement preStop hooks.
Karpenter Spot support: Automatic On-Demand fallback if Spot unavailable.
Spot Instance Advisor: AWS tool shows interruption frequency per instance.

Cost Savings

  • › Spot: 70-90% off On-Demand. E.g., t3.xlarge: $0.1664 → $0.05
  • › Typical interruptible workloads: batch, CI/CD, stateless apps
  • › With HPA + Karpenter: auto-provision Spot, fallback to On-Demand
Segment 6 — Right-Sizing with Goldilocks

Right-Sizing Workflow

Goldilocks + VPA

1. Install VPA — collects resource usage over time
2. Install Goldilocks — provides dashboard and recommendations
3. View recommendations — CPU/memory requests by pod
4. Apply recommendations — update deployment manifests
5. Measure impact — cost down, performance stays same

Iteration Cycle

  • › Analyze → Set requests → Measure → Iterate
  • › Run for 2-4 weeks before finalizing
  • › Seasonal workloads: right-size per season
Quiz

Cost Optimization Quiz ⚡

Q1. You have stable 80% baseline load. Should you use On-Demand or Reserved Instances?
✅ Reserved Instances. For stable, predictable load, RI gives 70% discount vs On-Demand.
Q2. A pod requests 4 CPU but only uses 1 CPU on average. Impact on cluster cost?
✅ Wasted capacity. That 3 CPU slot can't be used by other pods. Correct requests down to ~1.2 CPU.
Q3. Spot instances cost $0.50/hr vs On-Demand $1.50/hr. What's the downside?
✅ Interruption risk. AWS can reclaim Spot with 2-min notice. Use for fault-tolerant stateless workloads only.
Q4. AWS Compute Optimizer recommends t3.large (current: t3.xlarge). What should you do?
✅ Test in staging first. Verify performance stays within SLO. Then apply to production if safe. Saves ~30%.
Q5. You enable AWS Cost Anomaly Detection. It alerts: "Spend up 25%". Next step?
✅ Investigate: check Cost Explorer by service. Likely causes: new deployment, untagged resource, misconfiguration. Root cause, then fix.
