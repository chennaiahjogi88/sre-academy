Segment 1 — Infrastructure

Cloud vs On-Premise

🏢 Traditional (On-Premise)

  • Buy physical servers upfront (CapEx)
  • You manage hardware, power, cooling
  • Fixed capacity — hard to scale
  • Takes weeks to provision new servers
  • High maintenance cost

☁ Cloud (AWS)

  • Pay-as-you-go (OpEx) — no upfront cost
  • AWS manages physical infrastructure
  • Elastic — scale up/down in minutes
  • Provision resources in seconds
  • Global reach out of the box

💡 Analogy: Owning a car vs using Ola/Uber — you only pay when you need it

Segment 1 — Infrastructure

Cloud Service Models

🖥

IaaS

Infrastructure as a Service

  • You manage: OS, apps, data
  • AWS manages: hardware
  • Example: EC2
⚙

PaaS

Platform as a Service

  • You manage: apps, data
  • AWS manages: OS + infra
  • Example: Elastic Beanstalk, RDS
📱

SaaS

Software as a Service

  • You manage: your data only
  • AWS manages: everything else
  • Example: Gmail, Workdocs

💡 More control = more responsibility. Pick the model that fits your team's skills.

Segment 2 — AWS Global Infra

How to Choose a Region?

⚡

Latency

Choose the region closest to your end users. India users → ap-south-1 Mumbai

⚖

Compliance / Data Residency

Some regulations require data to stay within national borders (e.g., GDPR, India's DPDP Act)

💰

Cost

Prices vary ~20–30% between regions. us-east-1 is typically cheapest.

🛠

Service Availability

Not all 200+ services are available in every region on day 1. Check the AWS Regional Services list.

Segment 5 — Quiz

Rapid-Fire Quiz ⚡

Q1. What is an Availability Zone?
✅ An isolated data center (or group) within a Region, providing fault isolation.
Q2. Who is responsible for patching the OS on an EC2 instance?
✅ The Customer — EC2 is IaaS, so you manage the operating system.
Q3. Which AWS service would you use for serverless compute?
✅ AWS Lambda — runs your code without you managing any server.
Q4. What does S3 stand for?
✅ Simple Storage Service — used for scalable object/file storage.
Q5. Which service handles DNS routing in AWS?
✅ Amazon Route 53 — maps domain names to IP addresses.
